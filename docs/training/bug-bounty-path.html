Welcome to your journey to becoming a bug bounty hunter! This roadmap is designed to guide Kenyan learners and global InfoSecKe members through the essential stages — from learning basics to earning your first bounty.

📍 Phase 1: Learn the Basics

  • 🌐 Internet & HTTP Fundamentals
  • 🖥️ Operating Systems (Linux & Windows)
  • 🔐 Web App Basics (HTML, JS, Forms)
  • 📚 Study OWASP Top 10 → owasp.org

🛠 Phase 2: Master Tools

  • Burp Suite – Web traffic analysis
  • ffuf / dirsearch – Directory bruteforce
  • Nuclei – Automated vulnerability scanning
  • Subfinder – Subdomain enumeration

Install them via our Tool Setup Guide.

🧠 Phase 3: Hands-On Learning

  • 🧪 TryHackMe Labs → tryhackme.com
  • 🏁 PortSwigger Web Academy → Web Security Labs
  • 🎯 HackTheBox Machines → hackthebox.com

🪪 Phase 4: Join a Platform

These are the best bug bounty platforms to get started:

  • Bugcrowd
  • HackerOne
  • Intigriti
  • YesWeHack

📄 Phase 5: Learn to Report

Writing great vulnerability reports is key. A good report includes:

  • Title & severity
  • Step-by-step reproduction
  • Screenshot or video proof
  • Suggested fix

📝 Checklist

  1. ✅ Understand Web Tech & OWASP
  2. ✅ Master tools (Burp, Nmap, Nuclei, etc.)
  3. ✅ Solve 25+ web challenges (TryHackMe/HTB)
  4. ✅ Join platforms and read reports
  5. ✅ Submit valid vulnerability reports
